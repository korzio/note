Configure an access

  1. Create a Personal token
  2. Add it to the config file .githubrc to variable GITHUB_PERSONAL_TOKENSolution
      export GITHUB_PERSONAL_TOKEN=___TOKEN_GOES_HERE___
    1. Export this variable to the current shell with source command
      Solutionbash
      source config/.githubrc
  3. Use the auth key with @octokit/rest
  4. Get the list of Github issues

Install NPM dependencies

npm i cli-ux chalk @octokit/rest

Write the code

  1. Import cli from cli-ux to use advanced formatting
    Solution
      import cli from 'cli-ux'
    1. Import chalk from chalk to use colors
      Solutionts
      import chalk from 'chalk'
  2. Require the Octokit. This library is imported in a specific way
    import Octokit = require('@octokit/rest')
    1. Add a GITHUB_PERSONAL_TOKEN flag to flags definition so oclif will put the environment variable to a flag
      Solutionts
      githubPersonalToken: flags.string({
      description: `Environment variable GITHUB_PERSONAL_TOKEN`,
      env: 'GITHUB_PERSONAL_TOKEN',
      required: true
      })
  3. Use cli.action.start to show the loader with some useful information what is happening
    Solution
      cli.action.start('Getting the list of the issues')
    1. Create a new instance of Octokit with an object argument containing the "auth" property with the auth key created in the previous section
      Solutionts
      const octokit = new Octokit({
      auth: flags.githubPersonalToken
      })
  4. Call the "issues.listForRepo" method with an object argument containing "owner" and "repo" keys. You can pass "korzio" as an owner and "note" as a repository. Documentation of the method https://octokit.github.io/rest.js/#octokit-routes-issues-list-for-repo.
    The result of this method is an object containing "data" property
    Solution
      const { data: issues } = await octokit.issues.listForRepo({
    owner: 'korzio',
    repo: 'note',
  })
    1. Stop the loader with cli.action.stopSolutionts
      cli.action.stop()
  5. Show tha table with the "data" as the first argument and the object with table description as the second. You can use columns "title", "assignee" with a getter to get deep property, "state" with a getter to color the resulting state, "html_url" with a different header
    Solution
      cli.table(issues, {
    title: {


    },
    assignee: {
      get: row => row.assignee ? row.assignee.login : null,
    },
    state: {
      get: row => row.state === 'open' ? chalk.green('open') : chalk.red('closed'),
    },
    html_url: {
      header: 'Link'
    },
  })
